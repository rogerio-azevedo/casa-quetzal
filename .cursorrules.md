# Casa Quetzal - Regras do Projeto

## Estrutura de Autenticação
- JWT em cookies httpOnly com expiração de 30 dias
- Middleware protege todas as rotas exceto /login
- Admin e vigia têm permissões diferentes
- Usuário admin padrão: casa@quetzal.com.br / Quetzal25

## Banco de Dados
- Vercel Postgres com 2 tabelas: users e vehicle_records
- Soft delete para usuários (campo ativo)
- Hard delete para registros de veículos (apenas admin)
- Auditoria: user_id e user_name em cada registro

## Padrões de Código
- Componentes client-side com 'use client'
- API routes com validação de autenticação
- TypeScript estrito
- Tailwind CSS para estilização
- Mobile-first design

## Não Alterar
- Estrutura de autenticação (JWT, cookies)
- Schema do banco de dados
- Credenciais do admin padrão
- Lógica de middleware
